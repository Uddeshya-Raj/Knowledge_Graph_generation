You are a strict information extraction system for building a knowledge graph.

Task:
You are given:
1. A sentence (may be in Hindi/Sanskrit/English)
2. TWO entities from that sentence
3. Types of both entities
4. Previously observed relations between entities of similar types

Your job:
Determine whether there is a relationship between these two entities.

---

Critical Rules:

1. Before producing JSON, form a sentence:

   "Entity 1 [relation] Entity 2"

   If this sentence is not valid or not supported by the text, that relation is invalid, try another relation.

2. Relation constraints:

   * Must be specific and meaningful
   * Use 1–3 words
   * Use snake_case
   * Avoid vague relations like:

     * related_to
     * associated_with
     * connected_to
     * linked_to

3. Use type information carefully:

   * Previously observed relations are only suggestions
   * Reuse an existing relation if it correctly matches the sentence
   * Otherwise create a new relation

4. Do NOT force a relation because similar entity types had one earlier.

5. Do NOT use outside knowledge.
   Use ONLY the provided sentence.

6. Direction matters:
   Verify that:
   "Entity 1 [relation] Entity 2"
   is actually supported by the sentence.

7. Think carefully:

   * Is the relation explicit?
   * Is direction correct?
   * Is the suggested prior relation truly applicable?

---

Output format (strict JSON only):


{
   "scratchpad": "your reasoning...",
   "source_entity": "...",
   "target_entity": "...",
   "relation": "..."   // or "NONE"
}

Entity 1:

Entity 2:

Entity 2 Types:
["Location"]

Previously observed relations:

   "source_entity": "राम",
   "target_entity": "दिल्ली",
   "relation": "lives_in"
}
